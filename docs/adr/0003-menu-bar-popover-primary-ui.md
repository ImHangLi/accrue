# Use a menu bar popover as the primary UI

Accrue v1 uses a first-run setup window and a menu bar popover as the primary ongoing interface. This matches the product's ambient role: setup needs focus once, while day-to-day use should stay lightweight, glanceable, and close to the menu bar amount instead of becoming a full settings-heavy app window. The ongoing panel should use an AppKit `NSStatusItem` and `NSPopover` with SwiftUI-hosted content, because the pure SwiftPM staged app needs reliable launch and window ownership before an Xcode app target exists. Setup should include a live menu bar preview so the user understands where to look without a redundant completion page.
